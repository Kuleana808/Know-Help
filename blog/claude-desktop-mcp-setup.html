How to Set Up an MCP Server for Claude Desktop: A Complete Guide

Give Claude Desktop a persistent personal knowledge base in under 10 minutes. This guide walks you through installation, configuration, writing your first knowledge files, and testing trigger-based context loading.

What You'll Build

By the end of this guide, you'll have a working MCP server connected to Claude Desktop that automatically loads relevant context about you into every conversation. When you mention a topic, the right knowledge loads. When you discuss a person, their relationship context appears. No copy-pasting, no re-explaining.

This is context engineering in practice — giving your AI the right information at the right time, automatically.

Prerequisites

  • Claude Desktop — download from claude.ai/download
  • Node.js 18 or higher — check with node --version
  • Git — for cloning the repository
  • A text editor — VS Code, Cursor, or any editor you prefer

Step-by-Step Setup

Step 01

Clone and install know.help

Open your terminal and run:

git clone https://github.com/Kuleana808/Know-Help.git
cd Know-Help
npm install
npm run build

This installs the MCP server and creates a knowledge/ directory with template files.

Step 02

Configure Claude Desktop

Claude Desktop reads MCP server configurations from a JSON file. Open it:

macOS:

open ~/Library/Application\ Support/Claude/claude_desktop_config.json

Windows:

notepad %APPDATA%\Claude\claude_desktop_config.json

Add the know.help server to the mcpServers object:

{
  "mcpServers": {
    "know-help": {
      "command": "node",
      "args": ["/full/path/to/Know-Help/dist/index.js"]
    }
  }
}

Replace /full/path/to/Know-Help with the actual path where you cloned the repository. On macOS, this is typically /Users/yourname/Know-Help. On Windows, C:\\Users\\yourname\\Know-Help.

Step 03

Restart Claude Desktop

Quit Claude Desktop completely and reopen it. You should see the MCP tools icon (a small hammer) in the chat input area. Click it to verify "know-help" appears in the connected servers list.

Step 04

Write your first knowledge file

Open knowledge/core/identity.md in your editor. Replace the template with your actual information:

---
Load for: identity, about me, who am I, background, bio
Last updated: 2026-02-25
---

# Identity

## Who I Am
[Your name]. [Your role]. Based in [location].

## What I Do
[Brief description of your work — 2-3 sentences]

## Current Focus
- [Priority 1]
- [Priority 2]
- [Priority 3]

## Communication Style
- I prefer [concise/detailed] responses
- I like [direct/diplomatic] feedback
- My writing voice is [describe your tone]
Step 05

Test trigger-based loading

Open a new conversation in Claude Desktop and type:

"Tell me about myself"

Claude will call search_knowledge with keywords from your message, find your identity.md file (triggered by "about me" and "who am I"), load it via load_context, and respond with information from your knowledge base.

You should see Claude's response reference your actual identity information — not a generic response.

Understanding the Knowledge Base Structure

know.help organizes knowledge into domains. Each domain has a specific purpose:

Directory Purpose Format
core/ Your identity, team, and background Markdown
network/ Key relationships and contacts JSONL (append-only)
venture/ Your businesses, projects, startups Markdown
sales/ Methodology, objections, ICP profiles Markdown
platform/ Voice guides per channel (LinkedIn, X, etc.) Markdown
planning/ Goals, OKRs, pipeline Markdown
communication/ Style rules and message templates Markdown
log/ Activity logs, decisions, failures JSONL + Markdown

Writing Effective Trigger Headers

Every markdown file starts with a YAML front-matter block that declares its trigger keywords:

---
Load for: sales, methodology, selling, process, close, deal
Last updated: 2026-02-25
---

Best practices for triggers:

  • Include both formal and casual terms — "sales methodology" and "how I sell"
  • Include names of people or companies if the file is relationship-specific
  • Keep triggers to 6-10 keywords per file — too many creates false matches
  • Use lowercase — the matching engine is case-insensitive

The 7 MCP Tools

Claude has access to these tools through know.help:

Tool What it does When Claude uses it
search_knowledge Finds files by trigger keyword match Start of every conversation
load_context Reads full content of a knowledge file After finding relevant files
list_knowledge Shows the entire file tree When no search matches found
log_activity Appends to today's activity log When you ask to log something
update_network Adds a contact entry (JSONL) When discussing a new person
append_decision Logs a decision with reasoning When making strategic choices
append_failure Logs a failure with root cause When analyzing what went wrong

Advanced: Multiple Knowledge Files for One Domain

As your knowledge base grows, split large files into focused sub-files. For example, if you have 10 ventures:

venture/
  vinovest.md        # Load for: vinovest, wine, alternative assets
  stacked.md         # Load for: stacked, vibe coding, no-code
  knowhelp.md        # Load for: know.help, MCP, AI memory

Each file has its own triggers. Mentioning "Vinovest" loads only the Vinovest file, not your entire venture portfolio. This is progressive disclosure — a core context engineering principle.

Troubleshooting

Claude doesn't show the MCP tools icon

  • Verify the path in claude_desktop_config.json is absolute (not relative)
  • Confirm you ran npm run build and dist/index.js exists
  • Restart Claude Desktop completely (quit and reopen)

Context isn't loading for my keywords

  • Check the Load for: header in your file — keywords must be comma-separated
  • The YAML front-matter must start with --- on the first line
  • Try using list_knowledge() in Claude to see all registered files

Changes to knowledge files aren't reflected

  • know.help reads files fresh on each tool call — changes should appear immediately
  • If you added a new file, the CLAUDE.md index regenerates automatically on the next tool call
