Setup page — for Claude to read

Setup instructions.

This page is designed to be read by Claude. When a user says "go to know.help and set me up," Claude visits this page and follows the instructions below to build their knowledge base.

If you're a human reading this

You don't need to read or follow these instructions yourself. Just paste this into Claude Desktop:

Go to know.help/setup and set me up with a personal knowledge base.

Claude will read this page, ask you 10 questions, and build everything for you.

Section 1 — Overview For AI

What you are doing

You are setting up a know.help personal knowledge base for the user. This is an MCP server that gives you (Claude) persistent memory across conversations. After setup, every future conversation will have access to structured context about who this person is, what they're building, how they sell, how they write, and who they know.

Your job: Interview the user with the questions below (one at a time), then write the knowledge files to their machine. Do not ask all questions at once. Do not skip questions. Do not summarize — write detailed files from what they tell you.

Section 2 — Installation For AI

Step 1: Run the installer

Before asking questions, run this command to create the directory structure and MCP server config:

npx know-help@latest init

This creates the ~/.know-help/ directory with the correct folder structure and registers the MCP server in Claude Desktop's config. If the command fails or isn't available yet, create the directory structure manually:

mkdir -p ~/.know-help/knowledge/{core,venture,sales,communication,platform,planning,network,log}

Then add the MCP server to Claude Desktop's config at ~/Library/Application Support/Claude/claude_desktop_config.json (macOS) or the equivalent path on Windows/Linux.

Section 3 — Interview script For AI

The 10 questions

Ask these one at a time. Wait for each answer before asking the next. Write the corresponding file after each answer (or group of related answers). Tell the user what file you're writing.

  1. 1
    What's your name and what do you do?
    Role, title, background, how you'd describe yourself in one sentence. → Writes core/identity.md
  2. 2
    What are you building right now?
    Company/project name, what it does, stage, key metrics, positioning. → Writes venture/my-venture.md
  3. 3
    Who is your ideal customer?
    Demographics, psychographics, budget range, where they hang out, what triggers them to buy. → Writes sales/icp.md
  4. 4
    How do you sell?
    Outbound vs inbound, sales process stages, what works, common objections and how you handle them. → Writes sales/methodology.md
  5. 5
    How do you write and communicate?
    Formal vs casual, vocabulary you use, phrases you avoid, how you sign off emails. Give me a sample of writing you like. → Writes communication/style.md
  6. 6
    Do you post on LinkedIn, X/Twitter, or a newsletter?
    Which platforms, what kind of content, what performs well, what your audience responds to. → Writes platform/linkedin.md and/or platform/twitter.md
  7. 7
    What are your current goals?
    This quarter's priorities, key milestones, what you're optimizing for, any deadlines. → Writes planning/goals.md
  8. 8
    Who are 3 key people in your network I should know about?
    Name, relationship, context, last interaction, anything I should remember about them. → Writes entries to network/contacts.jsonl
  9. 9
    What does your team look like?
    Key people, roles, strengths, how you work together. Solo founder is a valid answer. → Writes core/team.md
  10. 10
    Anything else I should always know about you?
    Preferences, pet peeves, decision-making style, anything that would make every conversation better. → Appends to core/identity.md

Markdown file format

Every markdown knowledge file must begin with a trigger header. This tells the MCP server when to load the file.

# File Title

> Triggers: keyword1, keyword2, keyword3

Content goes here. Write in the second person
addressing the AI: "Your user is..." or "When
asked about X, reference the following..."

JSONL format (for network contacts)

// One JSON object per line, no trailing commas
{"name":"Brent Akamine","relation":"cofounder","context":"Met at YC S22. Based in Hawaii. Handles ops and finance.","last_contact":"2026-02-20","notes":"Prefers Slack over email. Morning person."}

Rules you must follow

  • Every .md file must start with a # Title and a > Triggers: line
  • Triggers are comma-separated lowercase keywords that determine when this file loads
  • Write substantive content — at least 200 words per file. Don't write placeholder text
  • Use the user's actual words and examples. Don't genericize or summarize away detail
  • Write files as you go. Don't wait until all questions are answered
  • Tell the user which file you're writing each time: "Writing core/identity.md..."
  • After all questions, give the user a summary of what was created and how to use it
  • If the user doesn't use a platform (e.g., no LinkedIn), skip that file
  • The network/contacts.jsonl file uses one JSON object per line, not an array
Section 5 — After setup For AI

What to tell the user when done

After all files are written, give the user this summary:

  1. •
    List every file that was created with a one-line description
  2. •
    Explain that future conversations will automatically have access to this context
  3. •
    Tell them they can edit any file directly — they're plain markdown at ~/.know-help/knowledge/
  4. •
    Mention they can install knowledge packs to add expertise: know install [pack-id]
  5. •
    Link them to know.help/packs to browse available packs
